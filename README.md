# Assignment 4: Graph Traversal and Representation System

## A. Project Overview
Данный проект посвящен реализации и анализу фундаментальных алгоритмов обхода графов (BFS и DFS), а также поиску кратчайших путей с использованием взвешенных ребер (Алгоритм Дейкстры) в качестве бонусного задания.
* **Структура графа:** Граф представлен в виде списка смежности (Adjacency List), реализованного через хэш-карту структур `Map<Integer, List<Edge>>`.
* **Вершины и Ребра:** Класс `Vertex` инкапсулирует уникальный числовой идентификатор узла.Класс `Edge` связывает исходную и целевую вершины, а также хранит вес ребра `weight` для корректной работы бонусного алгоритма.
* **Алгоритмы обхода:** Интегрированы алгоритмы BFS (обход в ширину) и DFS (обход в глубину).

## B. Class Descriptions
* **`Vertex`**: Модель узла графа, содержащая приватное поле `id`, конструктор, геттер и переопределенный метод `toString()`.
* **`Edge`**: Направленное ребро, содержащее ссылки на объекты `source` и `destination`, а также целочисленное значение `weight`.
* **`Graph`**: Основной управляющий класс структуры Хранит вершины и списки смежности ребер, предоставляет интерфейс добавления элементов (`addVertex`, `addEdge`), вывода структуры (`printGraph`) и выполнения алгоритмов поиска (`bfs`, `dfs`, `dijkstra`).
* **`Adjacency List`**: Использование динамических списков смежности позволяет оптимизировать занимаемую память до уровня $O(V + E)$, в отличие от матриц смежности, требующих фиксированного объема памяти $O(V^2)$.

## C. Algorithm Descriptions

### 1. Breadth-First Search (BFS)
* **Пошаговое описание:** Алгоритм использует структуру очереди (Queue) для итеративного обхода уровней графа по принципу FIFO. Хэш-множество `visited` предотвращает повторное посещение узлов и зацикливание. Соседние вершины текущего узла последовательно добавляются в конец очереди.
* **Сферы применения:** Поиск кратчайшего пути в невзвешенных графах, алгоритмы индексации поисковых роботов (Web Crawling), социальные сети (поиск друзей по уровням связей).
* **Временная сложность:** $O(V + E)$.

### 2. Depth-First Search (DFS)
* **Пошаговое описание:** Алгоритм осуществляет глубокое рекурсивное погружение по ветвям графа. При переходе к текущей вершине она помечается как посещенная, после чего метод рекурсивно вызывает сам себя для всех доступных несекретных соседей, пока не достигнет тупика, итеративно возвращаясь назад (backtracking).
* **Сферы применения:** Топологическая сортировка, поиск циклов в направленных и ненаправленных структурах, решение лабиринтов и головоломок.
* **Временная сложность:** $O(V + E)$.

## D. Experimental Results

### Таблица сравнения времени выполнения (в наносекундах)
| Размер графа (Вершины) | Время выполнения BFS (ns) | Время выполнения DFS (ns) |

| **30 Вершин** | 940,000 ns                | 702,800 ns                |
| **100 Вершин** | 2,570,700 ns              | 1,400,200 ns              |

### Наблюдения и закономерности
1. **Масштабируемость:** С ростом количества вершин (с 10 до 100) время выполнения обоих алгоритмов закономерно увеличивается, что подтверждает их линейную зависимость от размера входных данных.
2. **Сравнение скорости:** В данных экспериментах алгоритм DFS стабильно отрабатывает быстрее, чем BFS на всех размерах графов. Это связано с тем, что DFS использует встроенный стек вызовов Java (через рекурсию), требующий меньше накладных расходов памяти, чем итеративное создание объектов очереди `LinkedList` в BFS.

### Ответы на аналитические вопросы задания
1. **Как размер графа влияет на производительность BFS и DFS?** Время выполнения обоих алгоритмов растет линейно с увеличением количества вершин и ребер графа, так как алгоритмам приходится обрабатывать больший объем связей.
2. ** Какой обход оказался быстрее в экспериментах?** DFS завершается незначительно быстрее, поскольку рекурсивные вызовы в Java задействуют внутренний системный стек вызовов, что создает меньше накладных расходов в памяти, чем динамическое создание объектов `LinkedList` для очереди BFS.
3. **Соответствуют ли результаты ожидаемой сложности $O(V + E)$?** Да, результаты подтверждают теоретическую сложность. Наблюдается стабильное пропорциональное масштабирование времени без резких экспоненциальных скачков.
4. **Как структура графа влияет на порядок обхода?** Сильно связные и плотные графы заставляют BFS мгновенно расширять фронт волны на первом шаге, в то время как DFS уходит в глубокие единичные ветви. Разреженные структуры выравнивают поведение обоих алгоритмов.
5. **Когда BFS предпочтительнее DFS?** BFS незаменим, когда требуется найти кратчайший путь (минимальное количество шагов) или если искомый целевой узел расположен относительно близко к стартовой вершине.
6. **В чем заключаются ограничения DFS?** Главное ограничение DFS — риск возникновения критической ошибки переполнения стека (`StackOverflowError`) при работе с экстремально глубокими или бесконечными графами из-за глубоких уровней рекурсии.
